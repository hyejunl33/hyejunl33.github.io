# 프로젝트아이디어 - 에듀테크

## LMS(Learning Management System) **With AI**

교육청, 학교, 교사를 대상으로 하는 학급 관리 프로그램. 교사는 **OCR**된 학생의 포트폴리오를 이용해서 개인맞춤형 피드백을 제공할 수 있음.

## 0. 교육현장에서 AI 도구의 메리트

- 업무자동화
    - 많은량의 데이터를 한꺼번에 처리할 수 있음.
- 개인 맞춤형 교육
    - 다문화 학생에게도 맞춤형 교육을 제공
    - 느린학습자 지원
- 과정 중심 평가의 보조도구 → 결과중심평가에서 벗어날 수 있음
    - 2022 개정교육과정에도 부합함.
- 비대면 수업보조도구
    - 메타버스..? 실제로 코로나시기에 로블록스를 교실로 사용한 사례가 있음

## 1. 사례분석

### **1.1 구글 클래스룸**

![image](/assets/images/2025-11-21-20-05-18.png)
**장점**

- 아날로그 데이터 아카이빙
- 과제 제출여부, 제출시간등을 정량화해서 즉각적인 **정량평가 가능**

**단점**

- 데이터들을 엮어서 학생별 정성적 평가 불가능
- 과제는 단순한 사진 혹은 텍스트파일로 남음 → 정성적 평가 어려움

### 1.2 **다했니? 다했어요**

[https://edzip.kr/product-detail/682ed25b42cd392c8ffe1c8e](https://edzip.kr/product-detail/682ed25b42cd392c8ffe1c8e)

**장점**

- 구글 클래스룸과 마찬가지로, 과제평가, 과제 제출여부, 평가를 할 수 있음.
- 학생별로 포트폴리오를 아카이빙 할 수 있고 학부모와 공유하여 가정연계를 쉽게 할 수 있음.
- 강화를 통한 학생 동기부여
    - ‘쿠키’라는 강화방안을 통해서 학생들에게 동기를 부여함
        - 개선: 반별로 리더보드를 만들어서 더 동기를 부여할 수도 있을듯.

**단점**

- 과제 제출여부만 따지고 과제의 질적평가는 여전히 교사가 해야함.
- 비정형적인 데이터를 분석해서 자동화된 피드백을 제공할 수 없음
    - 일기, 독후감, 수학익힘책풀이등을 OCR한 후 채점보조를 해서 개선할 수 있음.

## 2. 문제의식

기존의 사례들에서는 학생들의 데이터를 아카이빙하는데에만 집중하고, **과제의 질적평가**는 여전히 교사의 몫이었다. 정량적 평가에서 **과정중심평가**로 교육의 패러다임이 바뀌고있음을 고려할때, 멀티모달 LLM을 이용해서 **정성적 평가**를 도울 수 있다.

## 3. 해결할 문제

AI가 대세다, 디지털 교과서를 사용한다 해도 결국에 교실에서는 **종이 책을** 이용해서 수업을 진행함 → 학생의 과제물들을 저장해서 AI모델을 이용한 정성적 평가에는  **OCR**이 필수적임.

### 3.1 휘발되는 각종 수업진행내용 → 이미지 아카이빙 혹은 OCR

교실에서는 기록할게 정말 많다..! → 학생의 교실에서의 생활 현황을 기록할 수 있음.

![image](/assets/images/2025-11-21-20-06-38.png)
![image](/assets/images/2025-11-21-20-06-44.png)
![image](/assets/images/2025-11-21-20-06-54.png)
![image](/assets/images/2025-11-21-20-07-00.png)

- **국어**: 글쓰기 활동 → OCR 후 학생 개인별 DB에 저장.
- **수학**: 수학익힘책 풀이→OCR을 이용해서 자동채점보조 → 학생 개인별 DB에 저장.
- **미술**: 만들기작품, 그림등을 아카이빙 → 정량적인 평가보단 교사의 정성적인 코멘트 → 해당 코멘트들을 모아서 학생의 개인화된 특징 파악
- **실과:** 만들기 작품(요리, 뜨게질) 사진으로 아카이빙 → 부모와 공유 및 교샤의 정성적인 코멘트 → 해당 코멘트를 모아서 학생의 개인화된 특징 파악
- **음악**: 리코더 및 단소와 같은 악기 및 가창 동영상으로 아카이빙 → 부모와 공유 및 교사의 정성적인 코멘트

아카이빙한 **학생별 포트폴리오**는 게시판형태로 **학부모**와 공유할 수 있음.

### 3.2 과제물 피드백할 시간부족 → OCR을 통한 채점 자동화

교사는 모든 학생들의 과제물(독후감, 수학익힘책등)을 읽고 채점해야한다. → 시간이 부족하다..

독후감, 수학익힘책, 과학등의 교과에서 OCR을 통해 채점을 자동화 한다.

### 3.3 학생별로 개별화된 피드백 제공

학생별로 아카이빙된 멀티모달 자료들을 바탕으로 학부모상담 자료, 생활기록부등을 생성할 수 있음.

## 4. 시장성

에듀테크 소프트웨어가 시장에 팔릴까? 

### 4.1 교사를 타겟팅

- 학급운영비
    - 교육청마다 다르지만 학급당 연간 20~50만원 수준
- 학급 환경 물품 구입비
- 학습준비물비

교사는 학급운영비로 소프트웨어를 구매할 수 있지만, 교육청별로, 학교별로 다르기 때문에 미지수임..

### 4.2 에듀테크 실증 지원사업

![image](/assets/images/2025-11-21-20-07-20.png)

프로젝트로 서비스개발 → 교사들에게 바이럴 → 사례를 모아서 에듀테크 실증 지원사업 공모전에 제출가능

### 4.3 교육청을 타겟팅

교육청을 타겟팅해서 소프트웨어를 팔려면 학교장터, 조달청에 등록을 해야한다..

- 사업자등록증 필요
- CSAP인증(보안인증)을 받아야함→ 딥시크OCR은 중국모델이라 힘들듯..

### 4.4 교과서 출판사를 타겟팅

![image](/assets/images/2025-11-21-20-07-31.png)

비슷한 서비스를 운영하는 미래엔이나 AI디지털교과서 출판에 관심있는 출판사에 exit을 노려볼 수 있음

## 5. 예상가능한 문제점

- 초등학생이 작성한 글이나수식에서 OCR한 결과가 만족할만한 성능이 나올까? → FInetuning, LoRA, 손글씨 인식을 잘하는 모델로 바꾸기
- 수학익힘책 채점을 자동화할때 풀이와 답안에는 정해진 양식이 없는데 어떻게 채점할것인가? 단순 OCR로 해결할 수 있는 문제인가?
- B2C가 아닌 B2B로 팔아야하는데, 과연 교육청이나 교사가 이용을 할만한 메리트가 있을까?
